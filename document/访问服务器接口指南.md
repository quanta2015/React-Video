# 访问服务器接口指南

部署完成后，您可以按照以下步骤访问服务器的 API 接口。

## 0. 快速测试（推荐）

### 方式一：使用签名脚本（最简单）

```bash
# 1. 设置 API_SECRET 环境变量
export API_SECRET=$(grep API_SECRET /opt/video-demo/nodejs/.env | cut -d= -f2)

# 2. 健康检查（不需要签名）
curl http://localhost:3000/health

# 3. 创建任务（使用签名脚本）
cd /opt/video-demo/nodejs
./scripts/sign-curl.sh --exec POST http://localhost:3000/api/tasks \
  '{"videoUrl": "https://example.com/video.mp4", "options": {"template": "t1"}}'
```

### 方式二：手动生成签名

```bash
# 1. 设置 API_SECRET
export API_SECRET=$(grep API_SECRET /opt/video-demo/nodejs/.env | cut -d= -f2)

# 2. 生成时间戳和随机数
TIMESTAMP=$(date +%s)
NONCE=$(openssl rand -hex 16)

# 3. 生成签名
BODY='{"videoUrl":"https://example.com/video.mp4","options":{"template":"t1"}}'
SIGNATURE=$(printf '%s' "${TIMESTAMP}${NONCE}${BODY}" | openssl dgst -sha256 -hmac "$API_SECRET" | awk '{print $NF}')

# 4. 发送请求
curl -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -H "x-timestamp: $TIMESTAMP" \
  -H "x-nonce: $NONCE" \
  -H "x-signature: $SIGNATURE" \
  -d "$BODY"
```

## 1. 确认服务状态

首先登录到服务器，检查服务是否正常运行：

```bash
# 查看 PM2 服务状态
pm2 status

# 查看服务日志
pm2 logs

# 查看 Nginx 状态
systemctl status nginx

# 查看 Redis 状态
systemctl status redis-server

# 查看 MySQL 状态
systemctl status mysql
```

所有服务应该显示为 `online` 状态。

## 2. 配置 API 密钥

### 2.1 自动生成 API Secret（推荐）

部署脚本会自动生成随机的 API Secret。如果你需要手动生成：

```bash
# 生成随机 API Secret（32 字节十六进制）
API_SECRET=$(openssl rand -hex 32)
echo "生成的 API Secret: $API_SECRET"

# 或者使用更简单的方式
API_SECRET=$(date +%s | sha256sum | cut -d' ' -f1)
echo "生成的 API Secret: $API_SECRET"
```

### 2.2 配置所有密钥

编辑 `.env` 文件，填入正确的密钥：

```bash
# 编辑环境配置文件
nano /opt/video-demo/nodejs/.env
```

需要填写的密钥：

#### 火山引擎 ASR 配置

```
VOLCENGINE_APP_ID=your_app_id
VOLCENGINE_ACCESS_TOKEN=your_access_token
VOLCENGINE_AI_API_KEY=your_api_key
```

获取方式：https://console.volcengine.com/speech

#### 阿里云 OSS 配置

```
ALIYUN_OSS_ENDPOINT=oss-cn-beijing.aliyuncs.com
ALIYUN_OSS_BUCKET=your_bucket_name
ALIYUN_OSS_ACCESS_KEY_ID=your_access_key_id
ALIYUN_OSS_ACCESS_KEY_SECRET=your_access_key_secret
ALIYUN_OSS_PREFIX=video-results
```

获取方式：https://oss.console.aliyun.com

配置完成后，重启服务：

```bash
pm2 restart all
```

## 3. 鉴权方式

API 使用 **HMAC-SHA256 签名**进行鉴权，不是简单的 API Key 方式。

### 3.1 签名算法

```
signature = HMAC-SHA256(timestamp + nonce + body, secret)
```

### 3.2 请求头要求

每个请求（除 `/health` 外）必须包含以下请求头：

| 请求头        | 说明                                 |
| ------------- | ------------------------------------ |
| `x-timestamp` | 秒级时间戳（10 位数字）              |
| `x-nonce`     | 随机字符串（建议使用 32 位十六进制） |
| `x-signature` | HMAC-SHA256 签名结果                 |

### 3.3 签名规则

1. **timestamp**: 当前 Unix 时间戳（秒），允许 ±5 分钟误差
2. **nonce**: 随机字符串，每个请求必须唯一（防重放攻击）
3. **signature**:
   - GET 请求：`HMAC-SHA256(timestamp + nonce + "", secret)`
   - POST 请求：`HMAC-SHA256(timestamp + nonce + body_json, secret)`

## 4. 访问 API 接口

### 4.1 本地访问（在服务器上）

#### 使用签名脚本（推荐）

```bash
# 设置 API_SECRET
export API_SECRET=$(grep API_SECRET /opt/video-demo/nodejs/.env | cut -d= -f2)

# 健康检查（不需要签名）
curl http://localhost:3000/health

# 创建任务
./scripts/sign-curl.sh --exec POST http://localhost:3000/api/tasks \
  '{"videoUrl": "https://example.com/video.mp4", "options": {"template": "t1"}}'

# 查询任务状态
./scripts/sign-curl.sh --exec GET http://localhost:3000/api/tasks/task_123

# 查询所有任务
./scripts/sign-curl.sh --exec GET http://localhost:3000/api/tasks

# 查询队列状态
./scripts/sign-curl.sh --exec GET http://localhost:3000/api/queue/status

# 查询可用模板
./scripts/sign-curl.sh --exec GET http://localhost:3000/api/templates
```

#### 手动生成签名

```bash
export API_SECRET=$(grep API_SECRET /opt/video-demo/nodejs/.env | cut -d= -f2)

# 生成签名参数
TIMESTAMP=$(date +%s)
NONCE=$(openssl rand -hex 16)
BODY='{"videoUrl":"https://example.com/video.mp4","options":{"template":"t1"}}'
SIGNATURE=$(printf '%s' "${TIMESTAMP}${NONCE}${BODY}" | openssl dgst -sha256 -hmac "$API_SECRET" | awk '{print $NF}')

# 发送请求
curl -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -H "x-timestamp: $TIMESTAMP" \
  -H "x-nonce: $NONCE" \
  -H "x-signature: $SIGNATURE" \
  -d "$BODY"
```

### 4.2 远程访问（从其他机器）

通过 Nginx 反向代理访问（端口 80）：

```bash
# 设置 API_SECRET（必须与服务器端相同）
export API_SECRET="your_api_secret"

# 健康检查（不需要签名）
curl http://<服务器 IP>/health

# 创建任务
./scripts/sign-curl.sh --exec POST http://<服务器 IP>/api/tasks \
  '{"videoUrl": "https://example.com/video.mp4", "options": {"template": "t1"}}'

# 查询任务状态
./scripts/sign-curl.sh --exec GET http://<服务器 IP>/api/tasks/task_123
```

**如果远程访问返回 404：**

1. 检查 Nginx 配置是否加载：

   ```bash
   nginx -t
   systemctl status nginx
   ```

2. 检查站点配置是否启用：

   ```bash
   ls -la /etc/nginx/sites-enabled/
   ```

3. 如果没有 `video-demo` 配置，重新加载：

   ```bash
   ln -sf /etc/nginx/sites-available/video-demo /etc/nginx/sites-enabled/
   nginx -t && systemctl reload nginx
   ```

4. 检查 Nginx 日志：
   ```bash
   tail -50 /var/log/nginx/error.log
   tail -50 /var/log/video-demo/nginx-error.log
   ```

## 9. 常见问题

### Q1: 连接被拒绝

```
curl: (7) Failed to connect to <服务器 IP> port 80: Connection refused
```

**解决方案：**

1. 检查 Nginx 是否运行：`systemctl status nginx`
2. 检查防火墙设置：`ufw status`
3. 确保端口 80 已开放：`ufw allow http`

### Q2: 502 Bad Gateway

**原因：** 后端服务未启动

**解决方案：**

```bash
pm2 restart all
pm2 logs
```

### Q3: 404 Not Found

**原因：** Nginx 配置未正确加载

**解决方案：**

```bash
# 检查配置是否启用
ls -la /etc/nginx/sites-enabled/

# 如果没有 video-demo 配置，重新启用
ln -sf /etc/nginx/sites-available/video-demo /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

### Q4: 401 Unauthorized

**可能原因：**

1. 缺少 `x-timestamp`、`x-nonce` 或 `x-signature` 请求头
2. 时间戳过期（超过 5 分钟）
3. nonce 已使用（重复请求）
4. 签名计算错误

**解决方案：**

1. 确保使用签名脚本或正确计算签名
2. 检查时间戳是否在当前时间 ±5 分钟内
3. 确保每个请求使用不同的 nonce
4. 验证签名算法：`HMAC-SHA256(timestamp + nonce + body, secret)`

### Q5: 任务一直处于 pending 状态

**原因：** Worker 服务未运行或队列堵塞

**解决方案：**

```bash
pm2 status
pm2 restart video-worker
pm2 logs video-worker
```

## 10. 签名脚本使用说明

签名脚本位于 `/opt/video-demo/nodejs/scripts/sign-curl.sh`

### 基本用法

```bash
# 设置 API_SECRET
export API_SECRET="your_secret"

# 生成签名 curl 命令（不执行）
./scripts/sign-curl.sh POST http://localhost:3000/api/tasks '{"videoUrl":"..."}'

# 直接执行
./scripts/sign-curl.sh --exec POST http://localhost:3000/api/tasks '{"videoUrl":"..."}'

# GET 请求
./scripts/sign-curl.sh --exec GET http://localhost:3000/api/tasks/task_123

# 从 stdin 读取 body
echo '{"videoUrl":"..."}' | ./scripts/sign-curl.sh POST http://localhost:3000/api/tasks
```

### 脚本原理

```bash
# 1. 生成时间戳
TIMESTAMP=$(date +%s)

# 2. 生成随机数
NONCE=$(openssl rand -hex 16)

# 3. 计算签名
SIGNATURE=$(printf '%s' "${TIMESTAMP}${NONCE}${BODY}" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $NF}')

# 4. 发送带签名的请求
curl -X POST "$URL" \
  -H "Content-Type: application/json" \
  -H "x-timestamp: $TIMESTAMP" \
  -H "x-nonce: $NONCE" \
  -H "x-signature: $SIGNATURE" \
  -d "$BODY"
```

## 11. 安全建议

1. **保护 API_SECRET**：不要将密钥提交到代码仓库
2. **配置 HTTPS**：生产环境必须使用 HTTPS 传输
   ```bash
   apt install certbot python3-certbot-nginx
   certbot --nginx -d your-domain.com
   ```
3. **限制访问 IP**：在 Nginx 中配置 IP 白名单
4. **定期更换密钥**：定期更新 `API_SECRET`
5. **监控异常请求**：检查日志中的 401 错误
