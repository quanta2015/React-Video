# 视频生成项目部署文档

本文档介绍如何将视频生成项目部署到云服务器。

---

## 部署架构

```
┌─────────────────────────────────────────────────────────────┐
│                      云服务器 (Linux)                        │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │
│  │   Nginx     │────▶│   Server    │────▶│   Worker    │   │
│  │  (反向代理) │     │ (Express)   │     │ (BullMQ)    │   │
│  └─────────────┘     └─────────────┘     └─────────────┘   │
│         │                   │                   │           │
│         ▼                   ▼                   ▼           │
│    80/443 端口        3000 端口          Redis 队列          │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │
│  │   HTTPS     │     │    MySQL    │     │    Redis    │   │
│  │   证书      │     │   数据库    │     │   消息队列   │   │
│  └─────────────┘     └─────────────┘     └─────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 方式一：手动部署（推荐新手）

### 步骤 1：准备云服务器

**推荐配置：**

- CPU: 4 核以上
- 内存：8GB 以上
- 磁盘：50GB SSD 以上
- 系统：Ubuntu 22.04 LTS 或 CentOS 8+

**云服务器提供商：**

- 阿里云 ECS
- 腾讯云 CVM
- 华为云 ECS
- AWS EC2

### 步骤 2：安装系统依赖

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y curl git ffmpeg mysql-server redis-server

# CentOS/RHEL
sudo yum install -y curl git ffmpeg mysql-server redis
```

### 步骤 3：安装 Node.js

```bash
# 使用 nvm 安装 Node.js 20 LTS
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
node -v  # 应显示 v20.x.x
npm -v   # 应显示 10.x.x
```

### 步骤 4：安装并配置 Redis

```bash
# 启动 Redis
sudo systemctl start redis
sudo systemctl enable redis

# 验证 Redis 运行
redis-cli ping  # 应返回 PONG
```

### 步骤 5：安装并配置 MySQL

```bash
# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 创建数据库和用户
sudo mysql -u root -p << EOF
CREATE DATABASE video_generator CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'video_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON video_generator.* TO 'video_user'@'localhost';
FLUSH PRIVILEGES;
EOF
```

### 步骤 6：克隆项目代码

```bash
cd /opt
git clone <your-repo-url> video-demo
cd video-demo/nodejs
```

### 步骤 7：安装项目依赖

```bash
npm install -g pm2  # 安装进程管理器
npm install         # 安装项目依赖
```

### 步骤 8：配置环境变量

```bash
# 复制环境配置模板
cp .env.example .env

# 编辑配置文件
nano .env
```

**必须配置的环境变量：**

```bash
# 火山引擎 ASR 配置
VOLCENGINE_APP_ID=your_app_id
VOLCENGINE_ACCESS_TOKEN=your_access_token
VOLCENGINE_AI_API_KEY=your_api_key

# 阿里云 OSS 配置
ALIYUN_OSS_ENDPOINT=oss-cn-beijing.aliyuncs.com
ALIYUN_OSS_BUCKET=your_bucket
ALIYUN_OSS_ACCESS_KEY_ID=your_access_key
ALIYUN_OSS_ACCESS_KEY_SECRET=your_secret
ALIYUN_OSS_PREFIX=video-results

# MySQL 配置
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=video_user
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=video_generator

# API 密钥配置（用于接口鉴权）
API_SECRET=your_random_secret_here

# 服务器配置
PORT=3000
```

### 步骤 9：初始化数据库

```bash
# 执行数据库初始化脚本
mysql -u video_user -p video_generator < schema.sql
```

### 步骤 10：使用 PM2 启动服务

```bash
# 启动 Server（API 服务）
pm2 start npm --name "video-server" -- run server

# 启动 Worker（视频渲染）
pm2 start npm --name "video-worker" -- run worker --concurrency 2

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

### 步骤 11：配置 Nginx 反向代理（可选）

```bash
# 安装 Nginx
sudo apt install -y nginx

# 创建配置文件
sudo nano /etc/nginx/sites-available/video-api
```

**Nginx 配置：**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

**启用配置：**

```bash
sudo ln -s /etc/nginx/sites-available/video-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 步骤 12：配置 HTTPS（推荐）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com
```

## 方式二：Docker 部署（推荐生产环境）

### 创建 Dockerfile

```dockerfile
FROM node:20-slim

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    ffmpeg \
    redis-server \
    default-mysql-client \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# 复制 package.json
COPY package*.json ./

# 安装依赖
RUN npm install --production

# 复制代码
COPY . .

# 构建 TypeScript
RUN npm run build

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["pm2-runtime", "start", "ecosystem.config.js"]
```

### 创建 docker-compose.yml

```yaml
version: "3.8"

services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    command: redis-server --appendonly yes

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: video_generator
      MYSQL_USER: video_user
      MYSQL_PASSWORD: video_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./schema.sql:/docker-entrypoint-initdb.d/schema.sql

  server:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - REDIS_HOST=redis
      - MYSQL_HOST=mysql
    env_file:
      - .env
    depends_on:
      - redis
      - mysql
    volumes:
      - ./output:/app/output
      - ./public:/app/public

  worker:
    build: .
    environment:
      - NODE_ENV=production
      - REDIS_HOST=redis
      - MYSQL_HOST=mysql
    env_file:
      - .env
    depends_on:
      - redis
      - mysql
    volumes:
      - ./output:/app/output
      - ./public:/app/public
    deploy:
      replicas: 2

volumes:
  redis_data:
  mysql_data:
```

### 启动 Docker

```bash
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 方式三：使用部署脚本（一键部署）

创建部署脚本 `deploy.sh`：

```bash
#!/bin/bash

set -e

echo "=== 视频生成项目一键部署脚本 ==="

# 1. 更新系统
echo "[1/8] 更新系统..."
apt update && apt upgrade -y

# 2. 安装依赖
echo "[2/8] 安装系统依赖..."
apt install -y curl git ffmpeg redis-server mysql-server nginx

# 3. 安装 Node.js
echo "[3/8] 安装 Node.js..."
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 4. 安装 PM2
echo "[4/8] 安装 PM2..."
npm install -g pm2

# 5. 启动 Redis
echo "[5/8] 启动 Redis..."
systemctl start redis
systemctl enable redis

# 6. 配置 MySQL
echo "[6/8] 配置 MySQL..."
mysql -u root << EOF
CREATE DATABASE IF NOT EXISTS video_generator CHARACTER SET utf8mb4;
CREATE USER IF NOT EXISTS 'video_user'@'localhost' IDENTIFIED BY 'video_password';
GRANT ALL ON video_generator.* TO 'video_user'@'localhost';
FLUSH PRIVILEGES;
EOF

# 7. 部署应用
echo "[7/8] 部署应用..."
cd /opt
if [ ! -d "video-demo" ]; then
    git clone <your-repo-url> video-demo
fi
cd video-demo/nodejs
npm install
cp .env.example .env

# 8. 启动服务
echo "[8/8] 启动服务..."
pm2 start npm --name "video-server" -- run server
pm2 start npm --name "video-worker" -- run worker --concurrency 2
pm2 save
pm2 startup

echo "=== 部署完成！==="
echo "API 地址：http://localhost:3000"
echo "查看状态：pm2 status"
```

执行部署：

```bash
chmod +x deploy.sh
sudo ./deploy.sh
```

## 常用运维命令

### PM2 管理

```bash
# 查看状态
pm2 status

# 查看日志
pm2 logs video-server
pm2 logs video-worker

# 重启服务
pm2 restart video-server
pm2 restart video-worker

# 停止服务
pm2 stop video-server
pm2 stop video-worker

# 删除服务
pm2 delete video-server
pm2 delete video-worker

# 查看监控
pm2 monit
```

### 日志查看

```bash
# 应用日志
tail -f /root/.pm2/logs/video-server-out.log
tail -f /root/.pm2/logs/video-worker-out.log

# 错误日志
tail -f /root/.pm2/logs/video-server-error.log
tail -f /root/.pm2/logs/video-worker-error.log

# Nginx 日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log
```

### 数据库管理

```bash
# 登录 MySQL
mysql -u video_user -p video_generator

# 备份数据库
mysqldump -u video_user -p video_generator > backup.sql

# 恢复数据库
mysql -u video_user -p video_generator < backup.sql
```

```bash
# 连接 Redis
redis-cli

# 查看队列
redis-cli LLEN bull:video-queue:wait

redis-cli FLUSHDB
```

## 安全建议

1. **修改默认密码**：修改 MySQL、Redis 默认密码
2. **配置防火墙**：只开放必要端口（80/443/3000）
3. **使用 HTTPS**：配置 SSL 证书
4. **API 鉴权**：配置 `API_SECRET` 环境变量
5. **定期备份**：备份数据库和 OSS 文件
6. **监控告警**：配置服务器监控和告警

## 故障排查

### 问题 1：Redis 连接失败

```bash
# 检查 Redis 状态
sudo systemctl status redis

# 查看 Redis 日志
sudo tail -f /var/log/redis/redis-server.log

# 重启 Redis
sudo systemctl restart redis
```

### 问题 2：MySQL 连接失败

```bash
# 检查 MySQL 状态
sudo systemctl status mysql

# 测试连接
mysql -u video_user -p video_generator -h localhost
```

### 问题 3：端口被占用

```bash
# 查看端口占用
sudo lsof -i :3000
sudo lsof -i :6379
sudo lsof -i :3306

# 杀死占用进程
sudo kill -9 <PID>
```

### 问题 4：FFmpeg 缺失

```bash
# 安装 FFmpeg
sudo apt install -y ffmpeg

# 验证安装
ffmpeg -version
```

## 性能优化

### 1. 调整 Worker 并发数

```bash
# 根据 CPU 核心数调整
pm2 restart video-worker -- run worker --concurrency 4
```

### 2. 配置 Swap（防止内存不足）

```bash
# 创建 4GB Swap
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 3. 优化 MySQL 配置

```bash
# 编辑配置文件
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 添加优化配置
[mysqld]
innodb_buffer_pool_size = 2G
max_connections = 200
```
